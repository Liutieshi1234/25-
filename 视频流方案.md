# 前端播放RTSP视频流+目标检测完整方案

## 一、整体架构

由于浏览器无法直接播放RTSP协议视频流，需要引入中间层进行协议转换。整体架构如下：

text

```
摄像头(或视频源) → ZLMediaKit(推流) → SpringBoot(拉流+目标检测) → ZLMediaKit(转推HTTP-FLV) → Vue(flv.js播放)
```



ZLMediaKit作为核心流媒体服务器，支持RTSP/RTMP/HLS/HTTP-FLV/WebSocket-FLV等多种协议之间的互相转换，同时SpringBoot后端负责拉流并进行目标检测处理。

## 二、Windows环境ZLMediaKit搭建

### 2.1 编译方式（推荐）

在Windows上使用ZLMediaKit有两种方式：直接使用预编译版本或自行编译。

**方式一：使用预编译版本（最简单）**

下载ZLMediaKit的Windows预编译版本，解压后双击`MediaServer.exe`即可运行，无需编译。也可以使用集成WebRTC功能的预编译包。

**方式二：自行编译**

需要准备以下环境：

- Visual Studio 2022（需安装C++桌面开发组件）
- CMake 3.10或更高版本
- Git客户端

编译步骤：

bash

```
# 克隆代码
git clone https://github.com/ZLMediaKit/ZLMediaKit.git
cd ZLMediaKit

# 初始化子模块（重要，否则编译会失败）
git submodule update --init --recursive

# 编译
mkdir build && cd build
cmake .. -G "Visual Studio 17 2022" -A x64
cmake --build . --config Release
```



### 2.2 启动ZLMediaKit

编译完成后，进入`release/Windows/Debug`或`release/Windows/Release`目录，执行：

bash

```
./MediaServer.exe -d  # -d表示以守护进程方式启动
```



### 2.3 推流测试（可选）

可以使用FFmpeg向ZLMediaKit推流进行测试：

bash

```
ffmpeg -re -i test.mp4 -vcodec h264 -acodec aac -f rtsp -rtsp_transport tcp rtsp://127.0.0.1/live/test
```



### 2.4 播放URL规则

ZLMediaKit中定位一个流媒体源主要通过四个元素：协议(scheme)、虚拟主机(vhost)、应用名(app)、流ID(streamid)。

对于RTMP类型的流媒体源，支持HTTP-FLV播放：

text

```
http://127.0.0.1/live/stream_id.live.flv
```



对于WebSocket-FLV：

text

```
ws://127.0.0.1/live/stream_id.live.flv
```



ZLMediaKit默认HTTP端口为80，可通过`config.ini`修改。

### 3.2 RTSP拉流+目标检测核心服务

java

```
import org.bytedeco.javacv.*;
import org.bytedeco.opencv.opencv_core.*;
import static org.bytedeco.opencv.global.opencv_imgproc.*;
import static org.bytedeco.opencv.global.opencv_imgcodecs.*;

@Service
public class VideoProcessService {
    
    // 目标检测服务（集成YOLO）
    @Autowired
    private YoloDetectionService yoloService;
    
    // ZLMediaKit推流地址
    private static final String PUSH_URL = "rtmp://127.0.0.1/live/detected";
    
    /**
     * 拉取RTSP流，进行目标检测，并推流到ZLMediaKit
     */
    public void processRTSPStream(String rtspUrl) {
        // 使用FFmpeg拉取RTSP流
        FFmpegFrameGrabber grabber = new FFmpegFrameGrabber(rtspUrl);
        grabber.setOption("rtsp_transport", "tcp");  // 使用TCP传输，更稳定
        grabber.setOption("stimeout", "5000000");    // 设置超时
        
        FFmpegFrameRecorder recorder = null;
        
        try {
            grabber.start();
            
            // 创建推流器，推送到ZLMediaKit
            recorder = new FFmpegFrameRecorder(PUSH_URL, grabber.getImageWidth(), grabber.getImageHeight());
            recorder.setVideoCodec(avcodec.AV_CODEC_ID_H264);
            recorder.setFormat("flv");  // 输出FLV格式
            recorder.setFrameRate(grabber.getFrameRate());
            recorder.setVideoBitrate(2000);
            recorder.start();
            
            Frame frame;
            while ((frame = grabber.grabImage()) != null) {
                // 将Frame转换为OpenCV Mat格式进行处理
                OpenCVFrameConverter.ToMat converter = new OpenCVFrameConverter.ToMat();
                Mat originalMat = converter.convert(frame);
                
                if (originalMat != null) {
                    // 执行目标检测，获取检测结果（带标注框的Mat）
                    Mat detectedMat = yoloService.detectAndDraw(originalMat);
                    
                    // 将处理后的Mat转换回Frame并推流
                    Frame outputFrame = converter.convert(detectedMat);
                    recorder.record(outputFrame);
                    
                    // 可选：通过WebSocket发送检测结果给前端
                    sendDetectionResultViaWebSocket(yoloService.getLastResults());
                }
            }
        } catch (Exception e) {
            e.printStackTrace();
        } finally {
            try {
                if (recorder != null) recorder.stop();
                if (grabber != null) grabber.stop();
            } catch (Exception e) {
                e.printStackTrace();
            }
        }
    }
}
```

### 3.4 WebSocket实时推送检测结果

java

```
@Component
@ServerEndpoint("/ws/detection")
public class DetectionWebSocket {
    
    private static CopyOnWriteArraySet<Session> sessions = new CopyOnWriteArraySet<>();
    
    @OnOpen
    public void onOpen(Session session) {
        sessions.add(session);
    }
    
    @OnClose
    public void onClose(Session session) {
        sessions.remove(session);
    }
    
    // 广播检测结果
    public void broadcastDetection(List<DetectionResult> results) {
        String json = JSON.toJSONString(results);
        for (Session session : sessions) {
            try {
                session.getBasicRemote().sendText(json);
            } catch (IOException e) {
                e.printStackTrace();
            }
        }
    }
}
```



### 3.5 Controller接口

java

```
@RestController
@RequestMapping("/api/video")
public class VideoController {
    
    @Autowired
    private VideoProcessService videoService;
    
    @Autowired
    private TaskExecutor taskExecutor;
    
    /**
     * 启动视频处理任务
     */
    @PostMapping("/start")
    public ResponseEntity<?> startProcess(@RequestParam String rtspUrl) {
        // 异步执行，避免阻塞
        taskExecutor.execute(() -> videoService.processRTSPStream(rtspUrl));
        return ResponseEntity.ok().body(Map.of("status", "started", "rtspUrl", rtspUrl));
    }
    
    /**
     * 获取ZLMediaKit播放地址
     */
    @GetMapping("/play-url")
    public ResponseEntity<?> getPlayUrl(@RequestParam String streamId) {
        // 返回HTTP-FLV播放地址
        String flvUrl = "http://127.0.0.1/live/" + streamId + ".live.flv";
        return ResponseEntity.ok().body(Map.of("flvUrl", flvUrl));
    }
}
```

### 4.2 Vue组件实现

vue

```
<template>
  <div class="video-container">
    <!-- 视频播放器 -->
    <video
      ref="videoPlayer"
      class="video-js vjs-default-skin"
      controls
      autoplay
      muted
      width="800"
      height="450"
    ></video>
    
    <!-- 检测结果展示 -->
    <div class="detection-panel">
      <h4>实时检测结果</h4>
      <div v-for="result in detectionResults" :key="result.id" class="detection-item">
        <span class="label">{{ result.label }}</span>
        <span class="confidence">置信度: {{ (result.confidence * 100).toFixed(2) }}%</span>
      </div>
    </div>
  </div>
</template>

<script>
import videojs from 'video.js';
import flvjs from 'flv.js';

export default {
  name: 'RtspPlayer',
  data() {
    return {
      player: null,
      detectionResults: [],
      ws: null,
      flvUrl: ''
    };
  },
  mounted() {
    this.initWebSocket();
    this.initVideoPlayer();
  },
  beforeDestroy() {
    if (this.player) {
      this.player.dispose();
    }
    if (this.ws) {
      this.ws.close();
    }
  },
  methods: {
    /**
     * 初始化视频播放器
     * 前端无法直接播放RTSP流，需要通过后端转换成FLV或HLS格式
     */
    async initVideoPlayer() {
      // 调用后端接口获取转换后的FLV播放地址
      const res = await this.$axios.get('/api/video/play-url', {
        params: { streamId: 'detected' }
      });
      this.flvUrl = res.data.flvUrl;
      
      // 使用flv.js播放HTTP-FLV流
      if (flvjs.isSupported()) {
        const flvPlayer = flvjs.createPlayer({
          type: 'flv',
          url: this.flvUrl
        });
        flvPlayer.attachMediaElement(this.$refs.videoPlayer);
        flvPlayer.load();
        flvPlayer.play();
      } else {
        // 降级方案：使用video.js播放HLS
        this.player = videojs(this.$refs.videoPlayer, {
          sources: [{
            src: this.flvUrl.replace('.flv', '.m3u8'),
            type: 'application/x-mpegURL'
          }],
          autoplay: true,
          controls: true
        });
      }
    },
    
    /**
     * 初始化WebSocket接收检测结果
     */
    initWebSocket() {
      this.ws = new WebSocket('ws://localhost:8080/ws/detection');
      this.ws.onmessage = (event) => {
        const data = JSON.parse(event.data);
        this.detectionResults = data;
      };
      this.ws.onerror = (error) => {
        console.error('WebSocket error:', error);
      };
    }
  }
};
</script>
```

### 4.3 前端播放方案对比

| 方案                      | 延迟         | 兼容性       | 实现难度 |
| :------------------------ | :----------- | :----------- | :------- |
| **HTTP-FLV + flv.js**     | 低(1-3秒)    | 不支持iOS    | 简单     |
| **HLS + hls.js/video.js** | 较高(5-10秒) | 兼容性好     | 简单     |
| **WebRTC**                | 极低(<500ms) | 需额外服务   | 中等     |
| **webrtc-streamer**       | 低           | 需部署中间件 | 中等     |

推荐使用HTTP-FLV + flv.js方案，延迟低且实现简单。如果对延迟要求极高（<500ms），可以考虑使用WebRTC方案。

## 六、常见问题与优化建议

### 6.1 ZLMediaKit配置优化

修改`config.ini`中的以下参数以提升性能：

- 启用`enable_rtp_tcp`：使用TCP传输RTSP流，更稳定
- 调整`http_flv_play_port`：HTTP-FLV播放端口（默认80）
- 配置`hls_seg_num`和`hls_seg_duration`：HLS切片参数

### 6.2 后端性能优化

- **帧采样**：目标检测不需要处理每一帧，可以设置间隔采样（如每5帧检测一次）
- **异步处理**：拉流和检测解耦，使用队列缓冲
- **对象池**：复用Mat对象，减少GC压力

### 6.3 前端兼容性

- iOS系统不支持flv.js，需要降级为HLS方案
- WebSocket连接断线需要自动重连
- 多路视频播放时注意浏览器性能限制

## 七、总结

本文提供了从ZLMediaKit搭建、SpringBoot后端拉流+目标检测、到Vue前端播放的完整解决方案。核心要点：

1. **ZLMediaKit**作为高性能流媒体服务器，支持RTSP推拉流和协议转换
2. **JavaCV + ONNX Runtime/YOLO**实现Java端的视频流处理和目标检测
3. **HTTP-FLV + flv.js**是前端播放RTSP转换流的最佳方案
4. **WebSocket**用于实时推送目标检测结果到前端展示
